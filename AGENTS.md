# AGENTS.md

## プロダクト概要
- このアプリは企業向けの会議室予約アプリ。
- 主な価値は「パソコンからすぐに予約できる手軽さ」と「迷わず操作できる高い操作性」。
- 単一企業向けの作り込みではなく、さまざまな企業で使いやすい汎用性を前提に設計する。

## 想定ユーザー
- 会議前に短時間で空き会議室を確保したい社員
- 複数会議室の利用状況をすばやく把握したい総務担当者や管理者
- 日常業務の中で説明なしでも使える業務アプリを求める企業ユーザー

## プロダクト方針
- 予約完了までの操作ステップはできるだけ少なくする。
- 空き状況の確認、時間選択、予約確定を最短導線で行えるようにする。
- 第一印象は現代的で洗練されていること。ただし装飾過多にはしない。
- 業務利用を前提に、視認性・一覧性・反応速度を見た目より優先する。
- PC利用を主軸に設計し、マウス操作でストレスなく扱える体験を重視する。

## UX原則
- ユーザーが最初に見る画面で、空き会議室と予約可能時間がすぐ分かること。
- 料金が発生する場合は、時間単位の金額がひと目で分かるように表示すること。
- 予約可能かどうかを色・ラベル・状態表示で明確に区別すること。
- 予約導線の途中で迷わせないこと。次に押すべき操作を常に明確にする。
- 入力項目は必要最小限に絞ること。長いフォームは避ける。
- 重要操作には即時フィードバックを返すこと。予約完了、競合、失敗は明確に伝える。
- 一覧と詳細の行き来を減らし、なるべく1画面内で判断・予約できるようにする。

## UIデザイン指針
- 雰囲気はモダン、クリーン、実務的。
- 情報設計は余白を活かしつつ、業務アプリとして密度不足にならないようにする。
- タイムテーブル、空き状況、予約ボタンなど主要要素の優先順位を明確にする。
- 配色は信頼感と視認性を重視し、企業利用で受け入れやすいトーンを選ぶ。
- アイコンや装飾は補助的に使い、情報理解を阻害しないこと。
- 状態変化は分かりやすく見せるが、過度なアニメーションは避ける。

## 機能優先順位
1. 空き会議室の一覧表示
2. 日時を指定した予約
3. 予約状況の視覚的な把握
4. 予約内容の確認・編集・キャンセル
5. 管理者向けの会議室管理や利用状況の確認

## 実装判断の基準
- 新機能追加時は「予約までの速さ」と「迷わなさ」を損なわないことを優先する。
- UI案に迷ったら、クリック数が少なく、一覧で判断しやすい案を採用する。
- データ表示は見た目の美しさより、業務上の判別しやすさを優先する。
- 料金表示は合計金額だけでなく、時間単位の価格が直感的に理解できる形を優先する。
- 将来的な企業ごとの差分設定を見据え、会議室情報や利用ルールは拡張しやすく保つ。
- 権限、予約競合、状態整合性など、業務アプリとしての信頼性を軽視しない。

## 避けること
- 初見ユーザーに説明が必要な複雑なUI
- 予約完了までに多くの画面遷移を要する設計
- モバイルアプリ的な演出をそのままPC画面に持ち込むこと
- 見た目優先で、空き状況や時間情報の読み取りやすさを下げること
- 企業ごとの個別要望に引っ張られ、汎用プロダクトとしての一貫性を崩すこと

## エージェントへの指示
- 提案・実装・デザイン判断は常にこのファイルの方針に従うこと。
- 仕様が未確定な場合は、操作性を優先した無難で実務的な案を採用すること。
- 画面設計では、まず「空き状況確認」と「即予約」の導線を最優先で成立させること。
- 文言、UI、機能案のすべてで、企業利用に耐える明快さと信頼感を維持すること。
- 重要な仕様判断や方向性の分岐でユーザーの意見が必要な場合は、その都度確認してから進めること。
- 複数の実装案やデザイン案に明確な差分がある場合は、独断で確定せず、選択肢を示してユーザーに聞き返すこと。
